Applying for health and social welfare benefits can be an administrative nightmare, especially for low-income families and elderly folks who need help the most. Too often, confusing rules and checklist barriers keep people from getting the support they deserve.

[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through how it works. (pause)
First, every query goes through our Security Checkpoint node, which scrubs out sensitive personal information like Social Security Numbers and blocks prompt injections.
If the request is clean, it is sent to the CareNavigator Orchestrator, which delegates work to specialized agents using AgentTool calls.
For eligibility questions, it routes to the Benefits Eligibility Specialist, which runs federal poverty guidelines math.
For applications and office lookups, it routes to the Application Guide Specialist.
Both specialized agents consult a local MCP Server to pull program rules, run income checks, and find administrative offices near your zip code.
Finally, the results go to a Human-in-the-loop checkpoint, which pauses the workflow to query the user for confirmation of necessary documents.
